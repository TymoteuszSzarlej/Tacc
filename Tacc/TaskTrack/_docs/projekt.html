Tacc Personal TaskTrack

System zarządzania zadaniami i czasem pracy

Opis ogólny

Nazwa Systemu: Tacc Personal TaskTrack

Cel systemu: Umożliwienie użytkownikom efektywnego zarządzania zadaniami, projektami oraz czasem pracy.

Zakres projektu: Webowa aplikacja w Django, dostępna z poziomu przeglądarki, z możliwością rozszerzenia o wersję mobilną.

Grupy docelowe

  • Freelancerzy
  • Studenci
  • Specjaliści IT
  • Osoby samodzielnie organizujące pracę

Dokumentacja wymagań

Wymagania funkcjonalne

  • Rejestracja/logowanie użytkownika
  • Tworzenie i zarządzanie wydarzeniami, zadaniami, projektami
  • Etykietowanie zasobów
  • Rejestracja czasu pracy (timery i manualne wpisy)
  • System powiadomień (np. przypomnienia)
  • Przegląd aktywności i statystyk
  • Panel administracyjny

Wymagania niefunkcjonalne

  • Bezpieczeństwo danych (autoryzacja, szyfrowanie)
  • Wydajność (praca w czasie rzeczywistym)
  • Skalowalność
  • Intuicyjny interfejs

Architektura systemu

Diagramy architektoniczne

System składa się z następujących komponentów:

  • Frontend: Interfejs użytkownika (HTML5, CSS, JavaScript)
  • Backend: Serwer aplikacji Django
  • Baza danych: PostgreSQL
  • System powiadomień: Celery + Redis

Opis środowiska technicznego

  • Framework: Django + Django REST Framework
  • Frontend: HTML5, CSS (Tailwind), JS (Alpine.js)
  • Baza danych: PostgreSQL
  • Hosting: Heroku / Railway / VPS
  • Dodatkowe narzędzia: Celery (do powiadomień i zadań cyklicznych), Redis (kolejki)

Opis technologii

Użyte technologie i narzędzia

Django Django REST Framework PostgreSQL Tailwind CSS Alpine.js Celery Redis Git + GitHub

Powody ich wyboru

  • Django: Szybki, solidny framework z dobrym wsparciem REST i wbudowanym panelem administracyjnym
  • PostgreSQL: Zapewnia wydajność, niezawodność i zaawansowane funkcje bazodanowe
  • Tailwind CSS: Umożliwia szybkie prototypowanie UI i tworzenie responsywnych interfejsów
  • Celery + Redis: Rozwiązanie do obsługi zadań asynchronicznych i powiadomień
